## 关于本书
前几天在网上看到一篇特别有意思的文章，内容是说10种让你在会议中看起来非常聪明的小把戏。本人作为一名咨（I）询（T）顾(min)问（gong），看万之后深感相见恨晚。后来才发现居然这10种把戏出自一本叫做 100 Tricks To Appear Smart in Meetngs的书。在咨询顾问这个行业中，使用书中描述的把戏的人一抓一大把。先不谈他们是真聪明还是像书中所说的只是在为了让自己看起来聪明。在会议中让自己看起来聪明过人，确实是可以吸引到比如Leader，PM之类的Key Person的注意，同时让自己有更多的机会。
引用作者的话就是

> The smart you appear, The more meetings youll be invited to. the more opportunities you'll have to appaer smart.

这就像是个良性循环。

## <a name="meeting"></a>关于会议
对于会议，作者的看法是会议的大部分时间都在浪费时间。但是我们要做的并不是改变会议的状态。而是在会议里尽可能的表现得聪明。这一点我是表示赞同的，与其浪费这些无意义的时间里，让自己看起来更聪明，然后给Leader留下个好印象。

首先作者提到在参加会议前的几点注意事项。

1. 进入到会议室后马上问有没有人需要什么？
2. 然后离开会议室，到处去消遣一下，做什么并不重要。
3. 带一些零食和饮料回来。
4. 坐在会议的Key Person的旁边，比如Leader，让大家觉得你好像也是会议的共同发起人。
5. 在白板上写一些Key Word
6. 和你的关系不好的同事进行眼神交流
7. 靠倒椅子双手抱头做出冥想的样子。

其实这里最重要的是**第4点**，就算半点关系都没有，坐在Leader旁边然后一副**【今天这个会议我们可是一起准备的啊】**的样子，不明真相的群众可是立马会上当的。

## <a name="trick"></a>一个个Trick
个人觉得作者所说的把戏不是每个都很有用。接下来就一个一个分析。感觉没用的就跳过。


- #### <a name="1"></a>会议中不断点头然后不断记笔记
点头表示你有跟上会议的内容，笔记内容根本不重要，就算是知识把发言者的内容原封不动的记录上，从旁观者来看也会觉得你是一个总结能力很强的人。记住，重要的不是内容，是别人眼中的自己！如果有人问你是不是在记会议笔记。一定要回答这是我个人用的笔记。这样你就不会露馅了！


- #### <a name="2"></a>用极慢的语速重复工程师说的最后一句话
通常工程师在会议中不发言，但是一旦工程师发言，有很多工程师都是站在开发角度说话，导致很多人听不明白工程师想表达什么。在工程师的神性发言结束后，在大家还没缓过神的时候，你只要几乎是原封不动的重复一遍工程师的发言后，再跟发言工程师确认这是不是他想表达的，这时候大家就会对你不明觉厉。看起来很简单的一件事，但是仔细想想，在这种自己还没跟上的时候，别人已经开始总结了，对这个人说的内容，更多的是一种恐慌的情绪，而不是试着去理解。
这里没有什么技术含量，都是愚蠢的人类的心理作用导致的！

- #### <a name="3"></a>叫发表的人返回上一张PPT
整个PPT的进程不重要，只要你说能返回上一张吗，大家会潜意识的觉得自己漏掉了什么，同时再一次被你的洞察力个吓尿。但是如果你自己没有任何的洞察怎么办？不用担心，你只要沉思一会，然后说OK，继续吧。大家反而会沉思你到底注意到了什么细节。不管有没有什么发现。尽管用！

- #### <a name="4"></a>开自己的玩笑，自嘲
自嘲是非常有用的，比仅仅是在会议里。这是人性问题，就和【有什么不开心的事情说出来让我开心开心】是一个道理。当有人问你对刚才一个小时的会议有什么意见的时候。假如你根本没有跟上，做一个耿直Boy，【**抱歉，刚才的一个小时我没有听见有人说话**】。大家也知道会议说的很多都是废话，对于的勇气和幽默表示敬佩。至少在我的职场里，这中耿直Boy还是不会被为难到的。

- #### <a name="5"></a>装出一副你知道一切的样子
这一招非常的万能，但是也很难，因为不懂装懂本来就是一门学问，而且对于咨（I）询（T）顾(min)问（gong）来说，每一次进新的项目，基本上很多东西都要重新学。很难有游刃有余的机会。我个人的建议是还是要大胆的用相关领域的专业词汇。因为其实和你对话的人，很有可能也是在不懂装懂。（不要怀疑，这种情况真的很多）如果是真的非常懂的人，说几句你就能看出来了，这个时候还是少说两句为妙。

- #### <a name="6"></a>说一些别人无法反驳的话
其实说的都是废话，但是大家只能对你说Yes，要知道在浅意识里，如果你经常对一个人表示认同。会加强此人所有发言对你的影响。

- #### <a name="7"></a>把数据比喻成易懂的内容
在会议中如果有人说了一个非常大的数据，可以把数据比喻成城市的人口或者GDP或者某个非洲国家的国土面积，这样大家会被你的学识渊博吓尿。

- #### <a name="8"></a>在会议的结束的时候和某个人留下来单独讨论一些“问题”
被你留下的同时会对你产生一种莫名的“**我们之间有一种别人没有的关系**”的错觉。然而没有被留下来的同事则会有种漏掉了什么的错觉，同时觉得你和被留下来的同时比自己知道更多的信息。但其实你只要和留下来的同时讨论下今天中午吃什么。

- #### <a name="9"></a>边听人说话边高速的打字
记住，一定要超高速。一般这种情况发生在和你的下属的会议上。特别是1 on 1对坐的会议伤。和上司开会你还是好好听着吧。和你会议的下属一定会被你的高速并行工作能力震惊的。然后这边的屏幕上，只是一串串乱码。

- #### <a name="10"></a>说你一个数据驱动决策型人才
当有人来跟你扯事的时候，先自豪的告诉他你是一个数据驱动决策的人才，没有数据坚决不约。如果你的同事碰巧有数据，那就问他要更多的数据，如果他有更多的数据，叫他整理好数据以便决策，如果他整理好了，那么数据应该过时了，跟他要最新的数据。

- #### <a name="11"></a>发表的时候尽量拿着一张笔和纸
在进行PPT说明的时候，如果不是大场合，尽量拿着一张笔和纸，这表明你做了充足的准备。然后当上司说了什么的时候，假装用笔记下来的样子。这比起你傻站在那说[对，对，是的，是的]要强一百倍。

- #### <a name="12"></a>在整个发表里插入一些没用的slide
如果不是大场合，找一些以前发表过的资料里比较难懂，比较复杂的资料插入到本次发表的资料里，然后在发表的时候故意跳过，边跳过边说你们不用在意这这些内容。短短10张PPT的内容，被你做出了20张的感觉，参与者会觉得你为此准备了很多。

- #### <a name="13"></a>质疑现在的提问是不是一个正确的提问。
烧脑时刻来了，个人很喜欢这个Trick，在讨论中如果有个人问了一个你觉得莫名其妙的问题，没有必要回答，直接了当的问他**[你觉得你的问题是一个正确的问题吗？]**。如果有人反问你你觉得什么才是一个正确的问题，机会来了，说你刚刚问了一个。被自己的机智感动。

- #### <a name="14"></a>在讨论的时候问大家如果CEO会怎么想
在讨论的时候，要求大家一定要思考如果是我们的CEO会怎么想。这个也是人的潜意识的问题，大家被这么一说肯定会被你诱导的去思考的，但其实CEO才没有关心你们讨论的事情好吗？

- #### <a name="15"></a>当有个非常好的idea的时候，质疑它，然后说你是在充当Devil's Advocate
英语里的[Devil's Advocate](https://zhidao.baidu.com/question/1510977337769503060.html)说的是不随大流的人。在大家都为一个点子较好的时候提出反对意见，大家会佩服你的勇气，在进行了一轮讨论之后，告诉大家你只是在做Devil's Advocate，是为了让这个点子变得更好，大家会再一次被你的机智感动。

- #### <a name="16"></a>要求击掌
这样做一方面可以让同事觉得你是一个友善，积极的人。同时，这就像训练小狗伸出手一样。一旦你的同事对你伸出手了。在某种深层意识里，他就是在听从你的指挥。在莫名其妙的时间点上要求击掌，同事都会觉得莫名其妙，但是都会跟你击掌。

- #### <a name="17"></a>在Business Dinner的时候，向服务员征求意见，然后点一道完全在Menu里不存在的菜
霸道总裁就是这样的，这种行为让你看起来就像是CEO一样。不管你们是吃日本菜，韩国菜，土耳其菜，这种做法都会让人觉得你非常熟悉这个菜系，好会享受生活，私生活一定很充实。

- #### <a name="18"></a>任何时候都保持sent from your iphone的签名
对于一些不重要的邮件，就算你不是在用手机，加上签名，就算你恢复的内容很粗糙同事也会理解你的，一只在奔波的你真的好忙。


整本书读起来还是很轻松的，有用的内容大概只有一半吧，感觉有点凑数的嫌疑。不过整本书的配图真的很到位，会让你很有临场感。在职场中，学习如何正确的装有时候比实力更重要。推荐进入职场3-5年的朋友们阅读和使用。刚刚加入公司的新人还是参考下就好了。
